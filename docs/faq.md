# Частые вопросы

## Зачем handoff если есть memory_search?

memory_search ищет по ВСЕЙ базе памяти - может вернуть фрагмент из прошлого месяца. handoff.md - это конкретно "что было в последней сессии". После компактификации агенту нужно знать не "всё что было за год", а "над чем работали 5 минут назад". handoff - это быстрый restore, memory_search - глубокий поиск.

## Можно ли без векторной памяти?

Да. Базовая система (handoff + daily notes + MEMORY.md + кроны) работает без векторного поиска. Агент будет полагаться на файловую память и handoff. Векторная память добавляет "умный поиск" - агент может вспомнить решение трёхмесячной давности по смыслу запроса.

## Сколько токенов жрут кроны?

Примерные расходы на Sonnet:
- auto-handoff (3x/день): ~500-1500 токенов на запуск, ~3000-4500 в день
- auto-diary (2x/день): ~500-1000 на запуск, ~1000-2000 в день
- consolidator (1x/неделю): ~2000-5000 на запуск
- night-cleanup: 0 токенов (bash скрипт)

Итого: ~4000-7000 токенов/день на кроны. На Sonnet это копейки (< $0.01/день).

## Как мигрировать с другой системы?

1. Скопируй существующие файлы памяти в `memory/core/`
2. Создай MEMORY.md - сводку ключевых фактов
3. Настрой кроны (handoff, diary)
4. Запусти `scripts/health-check.sh` - проверь что всё на месте

Если у тебя уже есть daily notes в другом формате - перенеси в `memory/`. Скрипт архивации разберётся.

## Почему MEMORY.md < 3000 символов?

MEMORY.md грузится в системный промпт КАЖДОЙ сессии. Чем больше файл - тем больше токенов тратится на каждый запрос. 3000 символов - это ~750 токенов, разумный баланс между "агент помнит важное" и "не жрёт лишнего".

Детали переноси в memory/core/ - они доступны через memory_search когда нужны.

## Что будет если удалить handoff.md?

Агент потеряет контекст текущей сессии при следующей компактификации. BOOTSTRAP.md попробует восстановить через sessions_history, но это менее надёжно. Не удаляй handoff.md.

## Можно ли использовать не OpenAI для эмбеддингов?

Да. В config/memory-search.json замени provider и model. OpenClaw поддерживает несколько провайдеров для эмбеддингов. text-embedding-3-small от OpenAI - просто самый дешёвый и быстрый вариант.

## Как часто обновлять MEMORY.md?

Еженедельный консолидатор делает это автоматически. Вручную - когда меняются приоритеты, ключевые решения или важные факты о владельце. Не нужно обновлять каждый день.

## Зачем черновики в handoff?

Самая частая потеря при компактификации - незаконченный текст. Ты пишешь пост, агент помогает, контекст сжался - и черновик пропал. Секция "Черновики" в handoff решает эту проблему: агент копирует незаконченный текст ЦЕЛИКОМ перед сжатием.
